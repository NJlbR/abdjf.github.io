# Черновечье — промо-сайт релиза

Статический сайт с двумя экранами:
- **Главная** с визуалом, треклистом и блоком платформ.
- **Карта мира** с канвас-анимацией, таймлайном и интерактивными точками лора.

## Краткий анализ проекта

- Архитектура максимально простая: `index.html` + `styles.css` + `script.js`, без сборки и зависимостей.
- Навигация между экранами реализована через хеш-маршруты (`#home`, `#map`).
- Интерактивная карта полностью рисуется в `<canvas>` и управляется конфигом сцен в `script.js`.
- Локальный запуск возможен через `server.mjs` (Node.js HTTP server).

## Локальный запуск

```bash
node server.mjs
```

После запуска открыть: `http://127.0.0.1:5173/`

## Релиз на GitHub Pages

В репозитории добавлен workflow `.github/workflows/deploy-pages.yml`, который:

1. Запускается при пуше в ветку `main` (и вручную через `workflow_dispatch`).
2. Публикует содержимое репозитория как статический сайт в GitHub Pages.

### Что нужно сделать в GitHub

1. Открыть `Settings → Pages`.
2. В секции **Build and deployment** выбрать **Source: GitHub Actions**.
3. Убедиться, что рабочая ветка — `main`.
4. Выполнить push в `main` (или вручную запустить workflow).

После успешного workflow сайт будет доступен по стандартному адресу:

`https://<username>.github.io/<repository>/`

> Если репозиторий называется `<username>.github.io`, тогда адрес будет `https://<username>.github.io/`.
